### OS 概述

####  知识总览

操作系统（operating system，os），是指控制和管理整个计算机系统的==硬件和软件==资源，并合理组织==调度==计算机的工作及资源的==分配==，以提供给用户和其他软件方便的==接口和环境==，是计算机系统中最基本的==系统软件==。

计算机系统的层次结构 ：裸机（纯硬件）、操作系统、应用软件、用户。

![](/images/os-notes/1.系统概述-20250526-183721.jpg)

![](/images/os-notes/1.系统概述-20250513-200428.jpg)

#### 操作系统是系统资源的管理者

执行程序需要将该程序放到内存中，才能被CPU处理

1. 处理器管理：进程控制、进程同步、进程通信、死锁处理、处理机调度	
2. 存储器管理：内存分配、地址映射、内存保护与共享和内存扩充	
3. 文件管理：文件存储空间的管理、目录管理及文件读写管理和保护
4. 设备管理：缓冲管理、设备分配、设备处理和虚拟设备	
5. 用户接口：

#### 向上层提供方便易用的服务

封装思想：将一些复杂的硬件功能封装成简单易用的服务，以方便用户

1. GUI：图形化用户接口	
2. 命令接口：联机命令接口（交互式）、脱机命令接口（批处理）
3. 程序接口：在程序中进行==系统调用==来使用，通过程序代码间接使用

命令接口和程序接口并称 “用户接口“ ，可由用户直接调用

#### 对硬件机器的扩展

1. 没有软件支持的计算机为裸机	
2. 把覆盖了软件的机器称为扩充机器，又称虚拟机

### OS 特征

#### 知识总览

![](/images/os-notes/1.系统概述-20250513-201059.jpg)

#### 并发性

计算机系统中“同时”运行多个程序，这些程序宏观上同时运行，微观上交替运行

1. 并发：指两个或多个事件在同一时间==间隔==内发生
2. 并行：指两个或多个事件在同一==时刻==同时地发生

操作系统伴随“多道程序技术”而出现，因此，操作系统和程序并发一起诞生

单核 CPU 同一时刻只能执行一个程序，各个程序只能并发地执行 
多核 CPU 同一时刻可以同时执行多个程序，多个程序可以并行地执行

#### 共享性

资源共享，指系统中的资源可供内存中多个并发执行的进程共同使用

1. 互斥共享方式：一个时间段内只允许==一个进程==访问该资源
2. 同时共享方式：允许一个时间段内由==多个进程==“同时”对资源进行访问

#### 虚拟性
 
 把一个物理上的实体变为若干个逻辑上的对应物

1. 空分复用技术：虚拟存储器技术
2. 时分复用技术：虚拟处理器

#### 异步性
  
多道程序环境中，允许多个程序并发执行，进程的执行以不可预知的速度向前推进

#### 特征间的关系

1. 并发性和共享性为最基本的特征，==互为存在条件==
2. 没有并发性，就谈不上虚拟性
3. 只有系统拥有并发性，才有可能导致异步性
4. 没有并发和共享，就谈不上虚拟和异步
5. 因此并发和共享是操作系统的两个最基本的特性

### OS 的发展和分类

#### 知识总览

![](/images/os-notes/1.系统概述-20250513-203039.jpg)

#### 手工操作阶段

主要缺点：用户独占全机、人机速度矛盾导致资源利用率极低

#### 单道批处理系统

引入==脱机输入/输出技术==，并由==监督程序==负责控制作业的输入、输出

1. 优点：缓解人机速度矛盾
2. 缺点：==内存中只有一道程序运行==，因而 CPU 有大量时间在空闲等待 I/O 完成

####  多道批处理系统（操作系统正式诞生）

每次往内存中读入多道程序
 
1. 优点：多道程序==并发==执行，共享计算机资源，资源利用率大幅提升
2. 缺点：没有人机交互功能，中途不能控制自己的程序执行

####  分时操作系统

计算机以==时间片==为单位轮流为各个用户/作业服务，用户可通过终端与计算机交互

1. 优点：用户请求可被即时响应，==解决了人机交互问题==，允许多个用户同时使用
2. 缺点：不能优先处理紧急任务，对各程序完全公平，循环地为用户服务一个时间片

#### 实时操作系统

系统能在严格时间内处理事件，具有==及时性和可靠性==

1. 优点:能优先处理紧急任务
2. 从可靠性看实时操作系统更强，从交互性看分时操作系统更强
3. 硬实时系统：必须在被控制对象规定时间内完成(火箭发射)
4. 软实时系统：可以松一些(订票)

#### 其他操作系统

网络操作系统：实现网络中各种资源的共享（如文件共享）和各个计算机之间的通信
 
 分布式操作系统：主要特点是==分布性和并发性==，系统中各台计算机地位相同，任何工作都可分布这些计算机上，由计算机并行、协同完成任务

个人计算机操作系统如 Windows XP、MacOS

### OS 运行机制

#### 指令

1. 指令是 CPU 能识别、执行的最基本命令，为二进制机器指令
2. Linux、Windows、MacOS 的命令窗口所使用的 ‘’指令‘’ 为 ‘’交互式命令接口‘’
3. ==特权指令==：操作系统内核作为“管理者”，可执行“特权指令”，如内存清零指令
4. ==非特权指令==：普通的运算指令，如：加法指令

#### CPU 的内核态、用户态

1. CPU 中有个寄存器：程序状态字寄存器（PSW），用1 表内核态，0 表用户态
2. 系统调用是 OS 提供给程序员可获得 OS 服务使得用户态变内核态的==唯一接口==
3. 内核态/核心态/管态：运行==内核程序==，执行特权指令
4. 用户态/目态：运行==应用程序==，执行非特权指令

#### 程序

1. 内核程序：操作系统的内核程序是系统的管理者，可执行特权（非特权）指令，运行在核心态。
2. 应用程序：为了保证系统能安全运行，普通应用程序只能执行非特权指令，运行在用户态。

#### 操作系统的内核

![](/images/os-notes/1.系统概述-20250502-174033.jpg)

内核能够执行特权指令，由内核划分功能的不同，==内核分为大内核和微内核==。

- 微内核
	1. 只负责最核心的一些工作。
	2. 优点：组织结构清晰，方便维护。
	3. 缺点：效率低。

- 大内核
	1. 会负责大部分的事情。
	2. 优点：效率高。
	3. 缺点：组织结构混乱，难以维护。

### 中断

#### 中断的概念及作用

中断（Interruption）是指计算机系统在执行正常程序流程时，由于内部或外部事件的发生，**临时暂停当前任务**，转而执行特定的处理程序（中断服务程序，ISR），处理完成后恢复原任务的过程。

“中断” 是==让操作系统内核夺回CPU使用权==的唯一途径。

1. 当中断发生时，CPU 立即进入核心态。
2. 当中断发生后，当前运行的进程暂停运行，并由操作系统内核对中断进行处理。
3. ==有了中断，才能实现多道程序并发执行==。
4. “中断”会使 CPU 由用户态变为内核态。

- 内核态 -> 用户态：
	1. 执行一条==特权指令==——修改 ==PSW== 的标志位为 “用户态”。
	2. 这个动作意味着操作系统将主动让出 CPU 使用权 。

- 用户态 -> 内核态：
	1. 由 “==中断==” 引发，==硬件自动完成变态过程==。
	2. 触发中断信号意味着操作系统将强行夺回 CPU 的使用权。

#### 中断的类型

中断信号的来源来自CPU内部称为内中断，外部称为外中断。

内中断：与当前执行的指令有关，中断信号来自 CPU 内部（异常、例外）
外中断：与当前执行的指令无关，中断信号来自 CPU 外部

1. 若当前执行的指令是非法的，则会引发一个中断信号
2. 若应用程序想请求系统内核的服务，执行==陷入指令==，引发内部中断信号
3. 执行“陷入指令”，意味着应用程序主动地将 CPU 控制权还给操作系统内核
4. “系统调用”就是通过陷入指令完成的

![](/images/os-notes/1.系统概述-20250513-204615.jpg)

#### 中断机制的基本原理

1. 不同的中断信号，需要用不同的中断处理程序来处理
2. ==中断处理程序一定是内核程序==，需要运行在 “内核态”
3. 当 CPU 检测到中断信号后，会根据中断信号的类型去查询“==中断向量表==”
4. 通过“中断向量表”可以找到相应的中断处理程序在内存中的存放位置

### 系统调用

#### 系统调用的概念

![](/images/os-notes/1.系统概述-20250513-205219.jpg)

1. 操作系统作为用户和计算机硬件之间的接口
2. 需要向上提供一些简单易用的服务，主要包括命令接口和程序接口
3. 其中，==程序接口由一组系统调用组成==

- “系统调用” 是操作系统提供给应用程序（程序员/编程人员）使用的接口
	1. 可以理解为一种可供应用程序调用的特殊函数
	2. ==应用程序可以通过系统调用来请求获得操作系统内核的服务==

#### 系统调用的作用

1. ==应用程序通过系统来调用请求操作系统的服务==（如存储分配、I/O 操作、文件管理等）
2. 操作系统内核统一掌管系统中的各种共享资源
3. 涉及共享资源的操作通过系统调用向操作系统内核提出服务请求
4. 由操作系统内核来调用共享资源
5. 这样可以保证系统的稳定性和安全性，防止用户进行非法操作

![](/images/os-notes/1.系统概述-20250513-205853.jpg)

#### 系统调用与库函数的区别

![](/images/os-notes/1.系统概述-20250513-205615.jpg)

编程语言中有很多库函数，部分是将系统调用封装起来，使上层进行系统调用更加方便。

#### 系统调用的必要性

==由操作系统内核对共享资源进行统一的管理==，并向上提供 “系统调用”。

1. 如果用户进程想使用如打印机这种共享资源，
2. 通过系统调用向操作系统内核发出请求，
3. 内核对各个请求进行协调处理。

#### 系统调用的过程

1. 通过==陷入指令传递内中断信号==引发中断
2. 转入相应的中断处理程序——系统调用的入口程序
3. 系统调用的入口程序根据==寄存器中的参数==判断用户需要哪种系统调用服务
4. 如指明系统调用类型 “fork”

![](/images/os-notes/1.系统概述-20250513-210352.jpg)

传递系统调用参数 → 执行陷入指令（==用户态==）→ 执行相应的内请求核程序处理系统调用（==核心态==）→ 返回应用程序

1. 陷入指令是在用户态执行的，执行陷入指令之后立即引发一个内中断
2. 内中断使 CPU 进入核心态 
3. ==用户态发出系统调用请求，核心态对系统调用进行处理==
4. 系统调用发生在用户态，对系统调用的处理发生在核心态
5. 执行陷入指令（自陷指令或访管指令）会处理内中断，使处理器（CPU）从用户态进入核心态。

### 体系结构

#### 知识总览

![](/images/os-notes/1.系统概述-20250513-211314.jpg)

#### 操作系统的内核

![](/images/os-notes/1.系统概述-20250502-174033.jpg)

==内核==是操作系统最基本、最核心的部分
==内核程序==是实现操作系统内核功能的程序

1. 操作系统内核需要运行在内核态 
2. 操作系统的非内核功能运行在用户态
3. 内核态下即可执行特权指令，也可执行非特权指令

![](/images/os-notes/1.系统概述-20250502-173903.jpg)

3. 而==变态的过程是有成本的==，要消耗不少时间，频繁地变态==会降低系统性能==

### 本章回顾

![](/images/os-notes/1.系统概述-20250513-212538.jpg)

1. 应用程序通过系统调用主动地请求操作系统的服务
2. 会使用“陷入指令/Trap/访管”主动引发一个异常，主动放弃 CPU 使用权

- CPU 为应用程序办事的过程：
	1. 可能会被管理者召回（中断）
	2. 也有可能是CPU自己检测到某些特殊情况（异常）
	3. 然后按照管理者的要求处理（重新回来为管理者办事）
