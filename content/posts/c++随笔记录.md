### 头文件与源文件

头文件（. H）与源文件（. Cpp）的分离是 C/C++传统模块化机制的核心，其根本目的在于将接口与实现解耦——头文件作为模块对外的“合约”，声明公开函数、共享数据结构（如结构体、枚举）乃至内部辅助定义；源文件则负责“履约”，具体实现这些功能。这种分工依赖编译链接机制：预处理时 `#include` 将头文件内容原样复制到每个包含它的源文件中，随后各源文件被独立编译为目标文件（. O 或. Obj），最后链接器将调用方与实现方匹配合并。尽管头文件的内容在每个包含它的源文件中都会产生编译时副本，但链接阶段会合并重复的类型定义并剔除无用代码，确保最终可执行文件干净无冗余。这种设计也带来了代价：主函数被迫接触其不关心的内部细节，导致信息过载；而修改头文件会触发所有包含它的源文件重新编译，造成编译耦合。然而，只要头文件的接口契约不变，修改实现源文件只需重新编译该文件即可，这在大规模开发中显著提升了构建效率。整体而言，这套机制以编译时重复和链接时优化为代价，换来了多文件协作的可行性与实现细节的封装性，是 C/C++构建复杂系统的基石。

### 类的本质


- C++ 的类本质上就是：
	1. C 的结构体 —— 数据容器
	2. 操作这些数据的函数 —— 方法
	3. 访问控制 —— public/protected/private
	4. 自动初始化/清理 —— 构造/析构函数

C++ 把 C 语言中需要手动组合的东西，通过语法糖和编译器支持，变成完整的"类"概念。

### 继承

继承是类与类之间的一种关系，允许一个类获取另一个类的数据及函数

- 子类对象在内存中就是一块==连续空间==：
	1. 前半段完整嵌入基类的所有成员变量
	2. 后半段为子类的新增部分
	3. 整块空间是一个整体，而不是使用指针/引用等指向概念

1. 复用代码：通过父类让子类的前半部分受到统一调控
2. 建立关系：建立 "is-a" 关系，子类是一种基类，基类指针可以统一操作所有子类对象，代码只需针对基类接口编写，就能自动适应未来新增的任何子类，而实现多态


> [!note] 继承的 is-a 关系
> 


1. is-a ：一个关系描述，用来表示继承关系，子类也是一种父类
2. B “is-a” A 时，意味着 B 继承自 A

编译器强制规定子类对象内存的前半段与基类完全兼容，并基于此在类型系统中开放了隐式转换，使得子类对象可以被安全地当作基类对象来指代和操作

- 静态调用（无虚函数）：`p->speak()`。编译器只看 `p` 的类型（`Animal*`），就直接去 `Animal` 类里找 `breathe` 函数。这是“is-a”的基础用法，完全在编译期决定。

- 动态调用（有虚函数）：`p->speak()`。编译器发现 `speak` 是虚函数，就不再死板地只看指针类型。它会去 `p` 指向的那段真实内存里，找到藏在对象头部的虚表指针 vptr，再通过 vptr 查到真正该调用的 `speak` 函数。此时，“is-a”的内存原理成为了跳板：基类指针先通过内存重叠找到对象，再利用对象里的 vptr 去溯源它的真实类型。
        

### 多态

多态：同一个接口，表现出不同的行为

| 多态  |               |             |        |
| :-: | :-----------: | :---------: | :----: |
| 运行时 |   虚函数 + 继承    |  运行代码时才决定   | 实际对象类型 |
| 编译时 | 函数重载、运算符重载、模板 | 编译代码时就已经决定了 |  变量类型  |


### 虚函数：运行时多态的实现机制


当基类与多个子类存在同名函数重写时，我们希望用一个统一接口处理所有这些关系类：
1. 于是我们利用继承的 "is-a" 关系，让父类指针可以接收所有子类对象
2. 但如果只用父类指针，调用的一律是父类版本
3. 为了让调用自动匹配到每个子类自己的实现，我们引入了虚函数

``` C++
void makeSound (父类 *p) {
    P->speak ();   //通过指针调用speak
} 
```



编译器为每个含虚函数的类生成一张虚函数表 vtable，里面按顺序存放所有虚函数的地址。每个对象内部隐藏一个虚表指针（vptr），指向自己所属类的虚表。

- 调用流程：
	1. 拿到基类指针 p
	2. 通过 p 找到对象的 vptr
	3. 通过 vptr 找到该类型的虚表
	4. 从虚表中取出对应函数的地址 
	5. 跳转执行

整个过程发生在运行期，但虚表本身在编译期就已生成。虚表就是一张提前写好的地址清单，让运行时不需遍历查找，直接按索引取地址——这是编译期对运行期的精准导航和提速优化。


### 纯虚函数、接口类

```C++
Virtual void speak () = 0;   //这就是纯虚函数
```

纯虚函数：函数在基类里不实现，强制要求所有子类必须自己实现

|  概念  |                         |
| :--: | :---------------------: |
| 纯虚函数 |    基类只声明，不实现，强制子类实现     |
| 抽象类  |     含有纯虚函数的类，不能实例化      |
| 接口设计 | 用抽象类定义“必须做什么”，子类决定“怎么做” |
